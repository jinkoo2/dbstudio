# DB Studio

Web-based SQL Server database explorer. FastAPI backend + single-page HTML/JS frontend.

## Stack

- **Backend**: Python 3.12, FastAPI, uvicorn, pyodbc
- **Frontend**: Single HTML file with vanilla JS, Cytoscape.js for ER diagrams
- **Database**: SQL Server via ODBC (configurable driver)
- **Config**: `.env` file (see `.env.example`)

## Project Layout

```
server.py                  # FastAPI app entry point, auto-generates data on startup
db.py                      # Shared DB config (loads .env) and connection helper
query_tables.py            # Queries schema -> data/tables.json
query_columns.py           # Queries schema -> data/columns.json
build_entity_relations.py  # Transforms tables.json -> data/entity-relations.json
query_patients.py          # Test script (standalone)
web/index.html             # Frontend SPA (ER diagram tab + SQL query tab)
web/serve.bat              # Windows launcher (conda activate + python server.py)
```

## Running

```bash
# Windows
conda activate dbstudio
python server.py

# Docker
docker compose up --build
```

Server runs on port 8080. Data files in `data/` are auto-generated on first startup if missing.

## Key Details

- ODBC driver name differs by OS: `SQL Server` (Windows default), `ODBC Driver 18 for SQL Server` (Linux/Docker). Set via `DB_DRIVER` env var.
- `POST /api/query` accepts connection credentials per-request from the frontend. The ODBC driver is read from env, not from the request.
- SQL query results are capped at 1000 rows (`MAX_ROWS` in server.py).
- Frontend stores connection fields (server, database, user) in `localStorage` key `dbstudio-connection`. Password is not persisted.
- `data/*.json` files may have UTF-8 BOM if generated by legacy C# tools. Python reads with `utf-8-sig` encoding to handle this.
- No test framework configured.
